# SG90 servo on starcam

Instructions for driving the SG90 micro-servo wired to **starcam** (Raspberry Pi B Rev 2, IP `192.168.4.28`).

## Hardware

SG90 wired to the Pi 40-pin header:

| Wire   | Pi header pin | Function           |
|--------|---------------|--------------------|
| Brown  | 6             | GND                |
| Red    | 2 or 4        | 5V                 |
| Orange | 12            | GPIO18 (PWM signal)|

Range: roughly ±90° from centre (180° total travel). Don't command past `value=±1.0`.

## Software requirements

**You must use the pigpio pin factory.** Other gpiozero factories fail on this old board:

- `lgpio` — can't open `/dev/gpiochip0` (user `peter` is not in the `gpio` group).
- `rpigpio` — refuses the Pi B Rev 2 "old-style revision code" (revision `000e`).
- Software PWM fallback — produces severe jitter, unusable for a servo.

pigpio uses DMA-timed pulses and is rock-steady regardless of CPU load.

The pigpio daemon and Python binding are already installed and enabled:

```bash
# Already done — for reference only:
sudo apt install -y pigpio python3-pigpio
sudo systemctl enable --now pigpiod
```

Verify before use:

```bash
ssh peter@192.168.4.28 "systemctl is-active pigpiod"   # should print: active
```

## Minimal Python example

```python
#!/usr/bin/env python3
import time
from gpiozero import Servo
from gpiozero.pins.pigpio import PiGPIOFactory

servo = Servo(18, pin_factory=PiGPIOFactory())

servo.value = 0.0    # centre
time.sleep(1)
servo.value = +1.0   # +90°
time.sleep(1)
servo.value = -1.0   # -90°
time.sleep(1)
servo.value = 0.0    # back to centre
time.sleep(0.5)
servo.detach()       # stop sending pulses; servo goes limp
```

`servo.value` is a float in `[-1.0, +1.0]` mapping to roughly `[-90°, +90°]`.

To convert an angle in degrees:

```python
servo.value = max(-1.0, min(1.0, angle_deg / 90.0))
```

## Smooth motion

Don't write large angle steps and sleep — the servo will jump and overshoot. To move smoothly, step in small increments at a fixed update rate (50 Hz is plenty) and clamp at the endpoints.

Reference script: `/home/peter/Berrylands/servo/smooth.py` on `pip`, or `/home/peter/smooth.py` on starcam itself. It sweeps -90° → +90° → -90° at 10°/s indefinitely; Ctrl-C stops it.

Pattern:

```python
import time
from gpiozero import Servo
from gpiozero.pins.pigpio import PiGPIOFactory

SIGNAL_PIN  = 18
DEG_PER_SEC = 10.0
STEP_HZ     = 50

servo = Servo(SIGNAL_PIN, pin_factory=PiGPIOFactory())
dt = 1.0 / STEP_HZ
step = DEG_PER_SEC * dt
angle, direction = 0.0, 1.0

try:
    t = time.monotonic()
    while True:
        angle += direction * step
        if angle >= 90.0:  angle, direction =  90.0, -1.0
        if angle <= -90.0: angle, direction = -90.0, +1.0
        servo.value = angle / 90.0
        t += dt
        sleep_for = t - time.monotonic()
        if sleep_for > 0: time.sleep(sleep_for)
        else: t = time.monotonic()
except KeyboardInterrupt:
    servo.detach()
```

## Running remotely from another host

```bash
scp myscript.py peter@192.168.4.28:/home/peter/myscript.py
ssh peter@192.168.4.28 "python3 /home/peter/myscript.py"
```

No `sudo` needed when using the pigpio factory — pigpiod runs as root and the Python client talks to it over a local socket.

## Stopping a running servo script

```bash
ssh peter@192.168.4.28 "pkill -INT -f myscript.py"
```

The script should catch `KeyboardInterrupt` and call `servo.detach()` to stop emitting pulses. A detached servo holds its position only mechanically — it has no torque.

## Conflicts to be aware of

A systemd unit `servo-clock.service` ("K2 Bus Servo Clock") exists on `deskpi` (a different Pi) and drives the same kind of servo there. It is **stopped and disabled**. There is no equivalent service on starcam — GPIO18 is free.

If anything else on starcam ever starts driving GPIO18 (PWM, audio, another script), expect fights over pulse timing. Check with:

```bash
ssh peter@192.168.4.28 "sudo lsof /dev/pigpio 2>/dev/null; pgrep -af 'gpio\|servo\|pwm'"
```

## Common mistakes

- **Forgetting `pin_factory=PiGPIOFactory()`** — gpiozero will silently fall back to software PWM and the servo will jitter wildly.
- **Driving past ±1.0** — strains the gearbox.
- **Not calling `servo.detach()` at the end** — pigpio keeps pulsing, which is mostly harmless but leaves the servo holding torque (and humming on cheap SG90s).
- **Powering from 3.3V** — SG90 needs 5V; use header pin 2 or 4.
- **Long unbroken sleeps after `servo.value = X`** — fine for a single move, but if you want smooth motion you must step in small increments.
