# 2026-06-17 — first half-day session

Set out to verify the previous night's transition (sun-altitude gate
in production for the first time), then build the experiments
machinery + the website player.

## What happened

### Morning: last night's data inspection

- Sun-altitude gate (-10°) **worked correctly** — clean transition
  at the right time, no flap cycle at the night-enter side.
- BUT -10° was too generous: first 7 frames after transition
  saturated (the 55s exposure caught real twilight even though sun
  was below -10°).
- Saw 26 daemon restarts through the rest of the night — the
  controller-side flip-back guard kept correctly killing the daemon
  when frames pegged, then re-entering night MODE_HOLD_TICKS later.
  Net effect: ~165 useful frames written.
- Decision: raise gate to **-14°** (commit `4db071f`). Also dropped
  the controller-side flip-back guard entirely (commit `899a90f`)
  on the principle that mode should be a pure function of sun
  altitude — brightness is observation, not control.

### Afternoon: deliverables + experiments

- Two saturation guards investigated and both fixed (commit
  `5e64f5a`): the streaming daemon's `saturation_stops=13.0` was
  unreachable in uint16; the controller's `streaming_v3w_lum()`
  didn't apply the saturation clamp the FITS path did.
- `diff-sweep` upgraded from `max(frame - mean)` per-frame to
  `max(frame) - K·mean - M·σ·√(2 ln N)` (commits `f7c49a3`,
  `2831efb`). publish-night-cam now uses M=1.0 (statistical noise
  floor cancelled), K=1.0 (principled mean subtraction).
- `bin/astro-experiment` + `astro/experiments.py` +
  `astro/experiments_impl/mci.py` landed (commit `92dea03`). One-off
  experiments now have a place to live alongside deliverables, with
  repo+commit traceability.
- Website player route at `/astro/<cam>/night/<night>/player`
  deployed (mywebsite commit `779c5cc`). Reuses the existing
  skycam advanced player (multi-source compare, frame-step, clip,
  share-URL).
- `bin/window-stack-sweep` ported from bash+heredoc to straight
  Python (commit `ee9a205`) after the 07:00 BST publish on puppy
  failed silently with `line 66: start: command not found` — a
  bash heredoc misfiring. Per Peter: "heredocs are flakey, bash is
  flakey, python is robust, re-useable script as part of the repo
  rock."
- `combined-brightness` stem-removal bug fixed (commit `0407cad`).
- Puppy pulled to current branch; rerun produced all three sweep
  mp4s under the new `eclipticam-v3w` S3 prefix.
